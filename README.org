** Description

Arduino container library similar to
[[http://www.cplusplus.com/reference/vector/vector/][std::vector]], but
backed by caller-provided static storage instead of dynamic allocation.

The container tracks a variable logical size while the maximum capacity comes
from the storage array you attach with the constructor or =setStorage()=.
That avoids heap usage on small embedded targets, but it also means your sketch
must provide valid backing storage before accessing elements.

This library pairs naturally with
[[https://github.com/janelia-arduino/Array][Array]]. =Vector= stores its
elements in external storage and does not need the maximum size as a template
argument. =Array= stores its elements internally and encodes the maximum size
in the type.

* Quick Start

** Basic setup

#+BEGIN_SRC cpp
#include <Vector.h>

constexpr size_t ELEMENT_COUNT_MAX = 5;
int storage[ELEMENT_COUNT_MAX];
Vector<int> vector(storage);

void setup() {
  vector.push_back(21);
  vector.push_back(42);
  vector.push_back(84);
}
#+END_SRC

You can also attach storage later:

#+BEGIN_SRC cpp
Vector<int> vector;
int storage[8];

void setup() { vector.setStorage(storage, 0); }
#+END_SRC

- =begin()= and =end()=

** Storage and bounds notes

- The default constructor leaves the container unattached to storage until
  =setStorage()= is called.
- =push_back()= is ignored when the vector is already full.
- =pop_back()= is ignored when the vector is empty.
- =remove(index)= only changes the container when =index < size()=.
- =operator[]()=, =at()=, =front()=, and =back()= do not perform bounds
  checking.

* Vector vs Array

** Vector

#+BEGIN_SRC cpp
constexpr size_t ELEMENT_COUNT_MAX = 5;
int storage[ELEMENT_COUNT_MAX];
Vector<int> vector(storage);
vector.push_back(77);
#+END_SRC

** Array

#+BEGIN_SRC cpp
constexpr size_t ELEMENT_COUNT_MAX = 5;
Array<int, ELEMENT_COUNT_MAX> array;
array.push_back(77);
#+END_SRC

Choose =Vector= when the storage already exists elsewhere or when you want the
capacity decided outside the type. Choose =Array= when the container should own
its fixed-capacity storage directly.
